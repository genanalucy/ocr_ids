# 第一期实施方案

## 任务边界

第一期输入已经裁切、方向正确的单个方块字图像，输出 IDS 树。版面检测、整行切字和第二期字形生成不在当前训练闭环内。

当前生产范围冻结为 `flat-structure-v1`：仅处理根节点为 `⿰`（左右）、`⿱`（上下）、`⿲`（左中右）或 `⿳`（上中下），且每个子节点都是叶部件的 IDS。包围结构、任何嵌套结构、截断字和多字图必须拒识或进入人工队列，不得作为一期模型的正例。

研究问题分为四级：

1. `closed`：字符见过，字体或退化条件未见；
2. `zero_char`：整字未见，全部叶部件见过；
3. `zero_combination`：部件和运算符见过，局部组合未见；
4. `zero_component`：出现未见部件，系统应拒识该区域而不是臆造答案。

第一篇可复现实验以 `zero_char` 的规范化树完全匹配率为主指标。

## 数据流水线

```text
CHISE/cjkvi/专家记录
        ↓
原始 IDS 解析与冲突检测
        ↓
版本化规范 IDS + 可接受替代答案
        ↓
多字体、多地区字形自动渲染
        ↓
按整字隔离的数据划分
        ↓
模型训练和开放集评估
```

训练标签以自动继承为主。人工工作集中在规范化规则、冲突记录和金标准测试集。

规划规模：首个 MVP 使用不少于 10,000 个整字和约 50 万张字体渲染图；之后扩大到可合法获得的全部 IDS 字符，并加入扫描体和手写体。

## 模型

主模型为 DINOv2 ViT 图像编码器加 6 层自回归 Transformer 解码器。第一版将先改为四类根结构分类加固定长度部件预测；解码只允许 `⿰`、`⿱`、`⿲`、`⿳` 和叶部件，从模型层面排除包围与嵌套输出。

辅助任务依次加入：

- 根结构运算符分类；
- 叶部件检索；
- 同一字不同字体的对比一致性；
- 笔画序列预测；
- 未知部件检测与置信度校准。

基线包括整字分类后查表、RAN 风格模型和无语法约束的视觉—文本 Transformer。

## 里程碑

### M0：基础设施

- IDS 解析、校验和树表示；
- JSONL 数据规范；
- 字体渲染器；
- 字符隔离的数据拆分；
- 单元测试和最小训练入口。

### M1：数据审计

- 导入 CHISE 和 cjkvi；
- 统计字符、部件、深度和运算符覆盖率；
- 输出冲突、循环、PUA/CDP 和地区字形报告；
- 冻结 `canonicalization-v1`。
- 执行 `flat-structure-v1` 过滤，并将超出范围结构单独保留。

### M2：模型基线

- 训练闭集查表基线；
- 训练 ViT + Transformer 基线；
- 完成 closed 与 zero-char 评估。

### M3：开放集增强

- 加入部件定位、笔画辅助和未知检测；
- 建立真实未编码字符金标准集；
- 输出错误类型和人工复核队列。

## 主要风险

- 同一码位地区字形不同，导致图像和 IDS 错配；
- 不同来源对“最小部件”的定义不同；
- 随机按图片拆分造成整字泄漏；
- 大型语言解码器记忆常见字符，掩盖视觉解析失败；
- 数据与字体许可证限制模型或数据集的发布方式。
